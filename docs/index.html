File Sorter for macOS

copper-golem

放り込んだファイルを、中身を読んで、ふさわしいフォルダへ。

Minecraft の銅ゴーレムが落ちたアイテムを同じ物のチェストへ仕分けるように、 監視フォルダに置かれたファイルを AI が内容から判断し、同じ階層のカテゴリフォルダへ静かに移動します。 合うフォルダが見つからなければ、無理に動かさずそのまま残します。

Python 3.11+ 外部ライブラリ依存ゼロ claude CLI を分類器に APIキー不要 MIT License
01

Concept

銅ゴーレムの挙動を、ファイル整理に翻訳する

銅ゴーレムは、近くに落ちたアイテムを拾い、同じ物がすでに入ったチェストへ優先して仕分けるモブです。 該当するチェストが無ければ、無理に押し込まず持て余します。この振る舞いをそのままファイルへ移しました。

銅ゴーレムcopper-golem
落ちているアイテムを拾う→監視フォルダ直下のファイルを拾う
同じ物が入ったチェストへ入れる→似た内容のファイルがあるフォルダへ移す
合うチェストが無ければ持て余す→合うフォルダが無ければ、その場に残す
斧で削れば元に戻せる→いつでも止められ、undo で巻き戻せる

「同一階層限定」が原則です。仕分け対象のファイルと、振り分け先のカテゴリフォルダは、同じ階層に並びます。

~/Downloads/inbox/ 請求書_2026.pdf ← ここに置く（分類対象） 請求書/ 契約書/ 写真/ ← 同じ階層の兄弟フォルダ ＝ 振り分け先 _未分類/ ← 合うフォルダが無いものの退避先（任意）
02

How it works

仕組み

設計の核心

Claude にファイルを触らせない。スクリプトが中身を取り出して文章として渡し、 claude -p は純粋な「分類器」としてだけ使う。 移動はすべてスクリプトが行う。だから速く、権限の扱いが単純で、安全。

Step 1 中身を抽出

拡張子に応じて先頭の数千字をテキスト化。ファイル名も必ず添える。

Step 2 候補を集める

兄弟フォルダの一覧と、各フォルダの説明（.golem.md か中のファイル名）をまとめる。

Step 3 AI が選ぶ

内容と候補を Claude（Haiku）へ。folder と confidence を JSON で受け取る。

Step 4 判定して移動

確信度が閾値以上なら移動し記録。未満なら「該当なし」として残す。

Claude が返すのは {"folder": "請求書", "confidence": 0.95, "reason": "…"} のような JSON です。 confidence が設定した閾値を下回るものは、フォルダ名が返ってきても「該当なし」として扱い、誤った振り分けを抑えます。

03

Features

機能

名前ではなく中身で判断し、安全に、そして見える形で仕分けます。

Content-based

中身で分類する

拡張子だけでなく、ファイルの内容そのものを読んで判断します。名前が曖昧でも内容で振り分きます。

  • テキスト・コード・CSV・JSON … 直接読み取り
  • PDF … 本文を抽出（pdftotext がある場合）
  • Word … テキストへ変換（textutil）
  • Excel（.xlsx / .xlsm）… シート名とセル文字列を抽出
  • 画像 … ファイル名と画素サイズ（任意で OCR）
No infra

AIインフラもAPIキーも要らない

すでに入っている claude CLI をそのまま分類器として呼び出します。サーバーもAPIキーも不要。 モデルは軽量・高速・低コストの Haiku で十分です。サブスク認証のまま動きます。

Always-on

即時＋定期の二段監視

ファイルを置いた瞬間に動きます（launchd の WatchPaths）。 さらに任意で一定間隔のスイープも併用でき、ダウンロード完了待ちで見送った分や、 後からフォルダを作った分を確実に拾い直します。 一度「該当なし」と判断したファイルは記憶し、内容が変わるか新しいフォルダが増えるまで再判定しません。 同じ通知を何度も出さず、Claude の呼び出しも繰り返しません。

Golem-faithful

合わなければ動かさない

どのカテゴリにも合わないファイルは、その場に残します。 まとめて退避させたい場合は、専用の退避フォルダ名を指定することもできます。

Tells you why

分けられない理由を知らせる

どのカテゴリにも合わずに残ったファイルは、各監視フォルダの _未分類レポート.md に 理由と確信度つきで一覧化されます。ライブ実行のたびに更新され、残りが無くなれば自動で消えます。 これを見て、新しいフォルダを作る・.golem.md を調整する判断ができます。

Safety first

安全機構を厚く

ファイルを勝手に動かす道具だからこそ、守りを固めています。

  • dry-run が既定（最初は移動せず判定だけ）
  • すべての移動を記録し、undo で巻き戻せる
  • 同名は上書きせず連番でリネーム
  • ダウンロード中・書き込み中のファイルは触らない
  • サブフォルダ自体は動かさない
Observable

結果が見える

ライブ実行で振り分け・残置・エラーが起きたとき、結果をデスクトップ通知でお知らせします （音つき・集中モードも越える）。dry-run では通知せず、何も起きなかった回は静かに何もしません。 ログは「移動した」「残した」「エラー」を明確に区別し、何か起きた回は件数を要約します。 ログを開かなくても状況が分かります。

Portable

設定だけで配布できる

監視先や挙動はすべて TOML の設定ファイルにまとまり、コードにパスを埋め込みません。 クローンして設定を置けば、誰のマシンでもそのまま動きます。

Zero deps

標準ライブラリのみ

Python 3.11 以降の標準ライブラリだけで動きます。pip での追加インストールは不要。 Excel の読み取りも、画像のサイズ取得も、外部パッケージなしで行います。

04

Decision

判定のしくみ

各ファイルは必ず「移動」「残置」「エラー」のいずれか一つに数えられます。 Claude の失敗（呼び出しエラーやタイムアウト）と、合うフォルダが無い「該当なし」を取り違えません。

[move]

候補のフォルダに合致し、確信度が閾値以上。実際にそのフォルダへ移動し、記録します。

[keep]

どのフォルダにも合わない、または確信度が低い。その場に残します（または退避先へ集約）。

[error]

claude の起動失敗・タイムアウト・出力不正、または移動の失敗。1件失敗しても処理は止まりません。

05

Getting started

使い方

まず手動の dry-run で振り分け先を確かめ、納得してから常駐させる、という順序を推奨します。

# 設定を用意して、監視フォルダや挙動を書く mkdir -p ~/.config/copper-golem cp config.example.toml ~/.config/copper-golem/config.toml # まず手動で確認（何も移動しない） python3 golem.py --dry-run [dry-run] 請求書_2026.pdf -> 請求書/ (conf=1.00) [keep] memo.txt (no match · conf=0.05 · …) # 納得したら、置いた瞬間に振り分ける常駐へ ./install.sh
# 直前の振り分けを元に戻す python3 golem.py undo # その場で全件スキャン（移動を実行） python3 golem.py --apply
# 常駐を解除する ./uninstall.sh # 開発者向け：テスト一式（83件） python3 -m unittest discover -s tests

振り分け先のフォルダは自分で用意します。監視ルート直下に 請求書/ 写真/ のように作るだけ。 フォルダに .golem.md を置いて「何を入れる場所か」を一文書くと、判定の手がかりになります。

# 各フォルダの中身から「何を入れる場所か」の説明を自動生成 python3 golem.py describe --dry-run [dry-run] 請求書/.golem.md # 請求書 ## 入れるもの 取引先に発行した請求書を保存するフォルダ。 ## 入れないもの 領収書、見積書 ## 手がかり 金額、支払期限、振込先、invoice python3 golem.py describe --apply # 実際に .golem.md を書き出す

describe は、フォルダにすでに入っているファイルの内容を読み、 .golem.md を「入れるもの／入れないもの／手がかり」の構造化テンプレートとして書き出します。 生成後は手で調整でき、「入れないもの」に除外条件を足すと、より意図どおりに分かれます。

06

Configuration

主な設定

~/.config/copper-golem/config.toml。変更は保存するだけで次回の実行から反映されます（再インストール不要）。

キー既定説明
watch_roots["~/Downloads"]監視するフォルダ。直下のファイルが対象。複数指定可。
modelclaude-haiku-4-5分類に使うモデル。軽量・高速・低コストの Haiku を既定に。
dry_runtruetrue の間は移動せず判定だけ表示。慣れたら false に。
on_no_match"keep"該当なしの扱い。"keep" で残す、フォルダ名で退避先へ集約。
confidence_threshold0.6この確信度未満は「該当なし」とみなす。誤分類の抑制。
stability_seconds20直近この秒数に更新されたファイルは見送る（DL中対策）。
interval_seconds0変化時に加え一定間隔でもスイープ。0 で即時のみ。
exclude_globs["*.crdownload", …]無視するファイル名パターン。未完了DLや一時ファイル。
enable_ocrfalse画像を OCR で読む（tesseract が必要・低速）。
notify / notify_soundtrue / trueライブ実行で何か動いたときにデスクトップ通知（dry-run では出ない）。音つき・集中モードを越える。
claude_bin""claude の絶対パス。空なら自動検出。常駐で見つからない時に指定。
report_file_未分類レポート.md残ったファイルと理由を書き出す名前。"" で無効。それ自体は分類されない。
07

Requirements

必要なもの

  • macOS常駐に launchd を使用。
  • claude CLIインストール済み・ログイン済み（サブスク認証で可）。これが分類の頭脳。
  • Python 3.11+標準ライブラリのみ。追加インストール不要。
  • terminal-notifier推奨。新しめの macOS では常駐からの通知に必要（無くてもログには残る）。
  • pdftotext / tesseract任意。あれば PDF 本文・画像 OCR も扱える。無くても劣化動作。
